“bcrypt is better than the normal crypto hashing functions like SHA-256 for storing passwords because it is specifically designed for password security.
First, bcrypt automatically adds a salt to each password, which ensures that even if two users have the same password, their hashes will be different. This protects against rainbow table attacks.
Second, bcrypt is intentionally slow and uses a cost factor. This makes brute-force attacks much harder because each password guess takes more time to process.
On the other hand, crypto hashing functions like SHA-256 are very fast and are designed for general-purpose hashing, not for password storage. Fast hashing is good for data integrity, but not safe for passwords.”

3. what is jsonwebToken  why we use or internal working 
👉.“JWT is a stateless authentication mechanism where the server generates a signed token after login. The client sends this token in future requests, and the server verifies it using a secret key.”

“jsonwebtoken (JWT) is used for authentication and secure data transmission between client and server. It generates a token after successful login, which the client sends with future requests to access protected routes.

A JWT has three parts: Header, Payload, and Signature.

The header contains the algorithm type, the payload contains user data like user ID, and the signature ensures the token has not been tampered with.

The server signs the token using a secret key, and when the client sends the token back, the server verifies it using the same secret key.”

5.what is session and JWT
👉
Sessions are stateful authentication, where the server stores user session data (usually in memory or database) and sends a session ID to the client. The client sends this session ID with every request.

JWT is stateless authentication, where the server does not store session data. Instead, all user information is stored inside the token itself, and the client sends that token with every request. The server verifies the token using a secret key.”
